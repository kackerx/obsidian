title: MVC和分层架构

- <mark class="hltr-pink">分层架构</mark>: 同类功能封装成一层的思想, 如entity层, model层, 持久化层(dao), 业务层(service), 表示层(jsp, php, go-template, handler, controller)
- 符合开闭原则, 想要新增功能只需在对应层新增代码, 而不去动以前的老代码
- web -> handler -> service -> dao

</br>

- <mark class="hltr-blue">MVC</mark>: 是一种软件架构的思想, 一种架构模式
