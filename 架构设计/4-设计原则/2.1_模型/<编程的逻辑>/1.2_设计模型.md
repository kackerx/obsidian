# 设计模型

````col
```col-md
~~~~ad-ex
title: 静态模型(类模型)

> <mark class="hltr-blue">静态模型</mark>: 关注系统的静态结构, 类名, 属性, 职责, 方法, 类之间的关系, 指导类的声明

</br>

- [<mark class="hltr-cyan">第一步</mark>] 参考领域类的映射
- 软件类 -> 领域类 -> 用例 -> 客户需求
- 筛选类, 提炼方法 需求用例中<mark class="hltr-pink">找动词</mark>, 根据动词分配给具体的类所属

</br>

- [<mark class="hltr-cyan">第二步</mark>] 应用设计原则和设计模式
- SOLID, 单一, 开闭, 里氏替换, 接口隔离, 依赖倒置
- 在设计原则的指导下, 应用设计模式
~~~~
```

```col-md
~~~~ad-tips
title: 动态模型(方法动作)

> <mark class="hltr-blue">动态模型</mark>: 描述类本身的动作或者状态变化, 以及类之间如何配合完成最终业务, 指导类的具体实现过程

</br>

- 状态模型
- 活动模型: 描述一个工作计算流程, 关注点在某项任务过程中, 对象承担了什么任务, 做了什么处理, 对象间的交互
- 序列模型: 时序图 or 顺序图, 
- 协作模型: 对象间关系
~~~~
```
````
